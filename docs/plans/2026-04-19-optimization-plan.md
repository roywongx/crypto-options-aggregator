# Crypto Options Aggregator: Direct Feature Integration Plan (FinceptTerminal Codebase Analysis)

> **Context:** This plan is generated by analyzing the actual source code structure of `FinceptTerminal` (C++/Qt) and mapping its precise functional patterns to be directly integrated into the Python/FastAPI architecture of `crypto-options-aggregator`.

## 1. Unified Event Bus (Inspired by `Fincept/src/datahub/`)
**Fincept Implementation:** Fincept uses a high-performance pub/sub `DataHub` architecture to route market ticks from WebSockets to UI components without blocking.
**Direct Integration for Crypto Options:**
- **Current Problem:** `crypto-options-aggregator` relies heavily on REST polling (`requests`) inside `ThreadPoolExecutor` (e.g., `_quick_scan_binance`). This causes latency and API rate limits.
- **Actionable Task:** Create `dashboard/services/datahub.py` using `asyncio.Queue` or `Redis Pub/Sub`. 
- **Implementation Details:**
  1. Create background tasks that maintain persistent WebSocket connections to Deribit and Binance.
  2. These WebSockets push tick data (Mark Price, Orderbook updates, DVOL) into the `DataHub` topics (e.g., `topic_btc_options`, `topic_dvol`).
  3. The `/api/quick-scan` endpoint no longer triggers network requests; it simply reads the latest state from the `DataHub` cache, dropping scan times from seconds to <10ms.

## 2. Exchange Abstraction Layer (Inspired by `Fincept/src/trading/brokers/`)
**Fincept Implementation:** Fincept supports 15+ brokers through a strict `BrokerInterface.h` polymorphism, allowing unified order matching.
**Direct Integration for Crypto Options:**
- **Current Problem:** `binance_options.py` and `deribit_options_monitor.py` have entirely different methods, forcing `main.py` to handle custom logic for each exchange.
- **Actionable Task:** Create an abstract base class `BaseExchange` in `dashboard/services/exchange_interface.py`.
- **Implementation Details:**
  - Define mandatory methods: `async get_options_chain()`, `async get_dvol()`, `async get_spot()`.
  - Refactor `binance_options` into `class BinanceExchange(BaseExchange)`.
  - Refactor `deribit_monitor` into `class DeribitExchange(BaseExchange)`.
  - **Immediate Benefit:** The `CalculationEngine` and `RiskFramework` can process data without knowing which exchange it came from. This paves the way to easily add **Bybit** or **OKX** options.

## 3. Paper Trading Engine (Inspired by `Fincept/src/trading/PaperTrading.cpp`)
**Fincept Implementation:** A built-in mock exchange engine (`OrderMatcher.cpp`) that simulates fills and tracks virtual portfolio PnL against live data.
**Direct Integration for Crypto Options:**
- **Current Problem:** The existing `/api/sandbox/simulate` is static. It only checks margin pressure at a specific moment in time.
- **Actionable Task:** Build a continuous Paper Trading engine for the `Sell Put` & `Rolling` strategy.
- **Implementation Details:**
  1. Add a SQLite table `paper_portfolio` tracking virtual cash (e.g., $50,000 USDT) and open positions.
  2. Add an endpoint `/api/paper/trade` to "open" a position based on current real-time premium.
  3. Run a background loop (matching Fincept's `OrderMatcher`) that calculates real-time Unrealized PnL (UPnL) and Margin Utilization based on live WebSocket mark prices.
  4. **Why this matters:** It allows you to test your "$55k/$45k Risk Framework" and "Rolling Down & Out" strategy safely over weeks of live market data without risking real capital.

## 4. MCP (Model Context Protocol) Integration (Inspired by `Fincept/src/mcp/`)
**Fincept Implementation:** Fincept exposes its internal market data to external AI models via the MCP protocol.
**Direct Integration for Crypto Options:**
- **Current Problem:** You are manually checking the aggregator and then asking AI (like me) for advice.
- **Actionable Task:** Add an MCP Server endpoint to the FastAPI app using the `mcp` Python SDK.
- **Implementation Details:**
  - Expose tools like `get_current_dvol()`, `get_highest_apr_put(strike, dte)`, and `calculate_roll_cost()`.
  - **Immediate Benefit:** Any MCP-compatible AI (Claude Desktop, Cursor, Gemini CLI) can directly connect to your running FastAPI server on `localhost:8080`, read the live crypto options data, and autonomously execute your trading strategy analysis.

## 5. AI Co-Pilot Chat Interface (Inspired by `Fincept/src/ai_chat/`)
**Fincept Implementation:** An embedded chat window (`AiChatScreen.cpp`) that uses `LlmService` to answer quantitative queries.
**Direct Integration for Crypto Options:**
- **Current Problem:** The "Advice Engine" is currently a static JSON output based on hardcoded rules.
- **Actionable Task:** Embed a chat widget in `index.html` communicating with a new `/api/copilot/chat` endpoint.
- **Implementation Details:**
  - Use the existing `agents/openai.yaml` prompt but hook it up to a live conversation endpoint.
  - The FastAPI backend intercepts the chat, injects the current `RiskFramework` status (e.g., "BTC is at $56k, near Regular Floor. DVOL is 45."), and sends it to the LLM.
  - The AI returns contextual answers like: "Because DVOL spiked, you can currently get 25% APR by rolling your 60k Put down to 55k. Shall I calculate the margin impact?"
